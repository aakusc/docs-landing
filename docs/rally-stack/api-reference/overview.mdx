## Overview

Rally Stack exposes a REST API for interacting with the platform programmatically. The API covers:

- **Modules** — query the module registry, validate compositions
- **AI Framework** — invoke ALMs, manage context, call the consensus engine
- **Authentication** — API key management and tenant authorization

## Base URL

```
https://rally.aicoderally.com/api/v1
```

For local development:

```
http://localhost:3000/api/v1
```

## Authentication

All API requests must include an `Authorization` header:

```
Authorization: Bearer <your-api-key>
```

API keys are scoped to a tenant and an access level (read, write, admin). See [Authentication](/api-reference/authentication) for details.

## Request Format

All POST/PATCH requests use JSON:

```
Content-Type: application/json
```

## Response Format

All responses return JSON:

```json
{
  "data": { ... },
  "meta": {
    "requestId": "req_01abc",
    "timestamp": "2025-03-01T12:00:00Z"
  }
}
```

Errors:

```json
{
  "error": {
    "code": "VALIDATION_ERROR",
    "message": "tenantId is required",
    "details": { ... }
  },
  "meta": { ... }
}
```

## Rate Limits

| Tier | Requests/minute |
|------|----------------|
| Studio | 60 |
| Edge | 300 |
| Summit | 1000 |

Rate limit headers are included in every response:

```
X-RateLimit-Limit: 300
X-RateLimit-Remaining: 287
X-RateLimit-Reset: 1740830400
```
