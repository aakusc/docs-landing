## Overview

This guide covers everything you need to start developing on Rally Stack. By the end, you'll have a working local environment, understand the repo layout, and know how to run, test, and create modules.

## Prerequisites

Before starting, install:

- **Node.js 18+** — use [nvm](https://github.com/nvm-sh/nvm) or [fnm](https://github.com/Schniz/fnm) for version management
- **pnpm** — `npm install -g pnpm`
- **Git** — standard git CLI
- A **Neon** account at [neon.tech](https://neon.tech) (free tier is fine for development)

## Clone and Install

```bash
git clone https://github.com/AICodeRally/rally-stack.git
cd rally-stack
pnpm install
```

pnpm workspaces will link all 130+ module packages locally. This takes 2–3 minutes on first install.

## Environment Setup

```bash
cp .env.example .env.local
```

Required variables:

```env
# Neon database (Three Laws: Neon required)
DATABASE_URL="postgresql://user:pass@host/dbname?sslmode=require"

# AI models
ANTHROPIC_API_KEY="sk-ant-..."    # required
OPENAI_API_KEY="sk-..."           # optional
GOOGLE_AI_API_KEY="..."           # optional

# Auth
NEXTAUTH_SECRET="generate-a-random-string"
NEXTAUTH_URL="http://localhost:3000"
```

## Database Setup

```bash
# Push schema to your Neon database
pnpm db:push

# Optional: seed with demo data
pnpm db:seed

# Open Prisma Studio to browse data
pnpm db:studio
```

<Tip>
  Create a dedicated Neon branch for your development: `neon branches create dev/your-name`. This gives you a full copy of the schema without affecting shared environments.
</Tip>

## Running the Dev Server

```bash
pnpm dev
```

This starts all packages in watch mode via Turborepo. The main app runs at `http://localhost:3000`.

For a specific app only:

```bash
pnpm --filter web dev
```

## Development Workflow

### Branch Strategy

```
main           → production
staging        → staging environment
feature/*      → your work
fix/*          → bug fixes
```

Create a branch before starting any work:

```bash
git checkout -b feature/my-module-name
```

And create a matching Neon branch:

```bash
# Using Neon CLI
neon branches create --name feature/my-module-name
```

### Making Changes

1. Identify which package your change belongs to
2. Run only that package in dev: `pnpm --filter @rally/package-name dev`
3. Write tests alongside your changes
4. Run `pnpm lint && pnpm typecheck` before committing

### Committing

```bash
# Stage specific files (never git add -A with modules — too many files)
git add packages/modules/my-module/...
git commit -m "feat(my-module): add commission calculator integration"
```

Follow [Conventional Commits](https://www.conventionalcommits.org/): `feat`, `fix`, `refactor`, `docs`, `test`, `chore`.

## Useful Commands

```bash
pnpm dev              # start all apps in dev mode
pnpm build            # build all packages
pnpm test             # run all tests
pnpm lint             # lint all packages
pnpm typecheck        # run TypeScript checks
pnpm db:push          # sync Prisma schema to database
pnpm db:studio        # open Prisma Studio
pnpm db:seed          # seed demo data
pnpm health           # run platform health checks
```
